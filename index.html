Deconstructing the Document Management Tool

This guide provides an interactive overview of the Kawasaki Document Management Tool (DMT). Explore the system's core components, user roles, and critical workflows to understand how it provides secure, role-based access to technical manuals and documentation.

Secure Logins

Internal Kawasaki users log in via Single Sign-On (SSO), while external customers use a secure email, password, and Multi-Factor Authentication (MFA) process.

Role-Based Dashboards

Each user role sees a customized dashboard. Admins see pending approvals and recent activity, while customers can pin important documents for quick access.

Conditional Approval Workflow

A key feature is the mandatory approval workflow, which acts as a quality gate whenever an action would alter a customer's view of documents.

Who Uses the DMT?

The system is built around five distinct user roles, each with a specific set of permissions and responsibilities. Select a role from the dropdown to visualize their access rights for key data types.

Permission Levels: 0 = No Access, 1 = Read, 2 = Read/Write

How It Works: Key Processes

The DMT operates on controlled workflows for managing users and content. The approval process is conditional—it's only triggered when an action could change what a customer sees.

Customer Onboarding Methods

1. Invitation Link

Product Support-Admin sends a secure, time-sensitive invitation link to a new customer contact, who then completes their own company setup.

2. Bulk Insert

Existing customers can be added in bulk by uploading a CSV file. The system validates entries against known company domain names.

3. Manual Creation

Admins or Product Support can directly create a new customer account by filling out a form within the DMT application.

Workflow 1: Admin Adds a Document

An approval is only required if the document's category is already linked to a customer, as this would immediately alter their view.
